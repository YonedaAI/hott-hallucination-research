### D2. SemType — Semantic Type (Source 1 Def 3.1; Source 2 Def 2.1; Hs:II)

A semantic type is a homotopy type A in a universe U where:
- **0-cells** (points a : A): valid propositions or claims of type A.
- **1-cells** (paths p : a =_A b): justifications or entailments between claims a and b.
- **2-cells** (α : p =_{a =_A b} q): equivalences between justifications.
- **n-cells**: higher coherence data ensuring all levels of justification are internally consistent.

Haskell encoding:
```haskell
data SemType where
  Entity   :: String -> SemType           -- atomic entity type
  Prop     :: String -> SemType           -- proposition type
  DepType  :: SemType -> (SemTerm -> SemType) -> SemType  -- dependent type (context)
  PathType :: SemType -> SemTerm -> SemTerm -> SemType    -- identity type a =_A b
  PathPath :: SemType -> SemTerm -> SemTerm -> SemType    -- higher path (homotopy)
  ProdType :: SemType -> SemType -> SemType
  SumType  :: SemType -> SemType -> SemType
  VoidType :: SemType                     -- absurdity / contradiction
  UnitType :: SemType                     -- trivially true
```

Key: `PathType` and `PathPath` are the carriers of higher-dimensional structure. `VoidType` is the type of contradictions — generating a term of VoidType is the formal definition of hallucination.

### D16. Knowledge Complex K_• (Source 1 Def 4.1; Source 2 Def 3.1)

Given a knowledge base K, the **knowledge simplicial complex** K_• is the simplicial set with:
- K_0: grounded atomic facts (0-simplices = sourced propositions)
- K_1: pairs of mutually consistent facts connected by a justification (1-simplices)
- K_n: (n+1)-tuples of mutually co-consistent facts with n-dimensional justificatory structure (n-simplices)

Face maps dᵢ: K_n → K_{n-1} omit the i-th fact.
Degeneracy maps sᵢ: K_n → K_{n+1} repeat the i-th fact.

### T1. Non-Faithfulness of F (Source 1 Prop 2.1)

**Proposition.** The functor F: Sem → Vect_R is not faithful: there exist distinct morphisms f, g: Γ → Δ in Sem such that F(f) = F(g) in Vect_R.

**Proof.** Two distinct justifications for the same conclusion (e.g., proving the Pythagorean theorem via similar triangles vs. area dissection) map to the same embedding region but are distinct 1-cells in the semantic ∞-groupoid with different higher-dimensional coherence data.

---

### T2. Contractibility of Vect (Source 1 Prop 2.2; Source 2 §1)

**Proposition.** The category Vect_R^fin, viewed as a topological space via its nerve, is contractible: π_n(|N(Vect_R^fin)|) = 0 for all n ≥ 0.

**Remark.** The ∞-groupoid completion of Vect_R^fin is contractible because every object is isomorphic to R^n for some n, and the automorphism groups GL_n(R) are connected. Every loop in embedding space can be contracted. No topological obstruction can be represented.

---

### T3. Non-Contractibility of Sem (Source 1 Prop 2.3)

**Proposition.** The semantic category Sem, viewed as an ∞-groupoid, generically has non-trivial homotopy groups:
```
π₁(Sem) ≠ 0,   H_n(Sem; Z) ≠ 0 for some n ≥ 1.
```

**Proof sketch.** The cyclic inference pattern "A because B, B because C, C because A" defines a loop (1-cycle) that cannot be contracted because no valid 2-cell (justification) fills it — it is circular reasoning. Hence π₁(Sem) contains a non-trivial element. The homological claim follows from the Hurewicz theorem.

---

### T4. Fundamental Obstruction Theorem (Source 1 Theorem 2.1; Source 2 Theorem 5.1)

**Theorem.** Let Sem be a semantic category with π_k(Sem) ≠ 0 for some k ≥ 1. Then there exists no faithful functor F: Sem → C where C is contractible.

**Proof.** Suppose F: Sem → C is faithful and C is contractible. The induced map on homotopy groups F_*: π_k(Sem) → π_k(C) = 0 sends every element to zero. If [γ] ≠ [e] in π_k(Sem), then γ and e (the constant loop) are distinct morphisms in the fundamental k-groupoid. Faithfulness requires F(γ) ≠ F(e). But F_*(γ) = F_*(e) = 0, so their images are homotopic, contradicting faithfulness at the morphism level.

**Corollary (Structural Inevitability).** Any language model architecture whose representation space is contractible (including all finite-dimensional vector space embeddings) will necessarily hallucinate on domains whose semantic structure has non-trivial homotopy groups.

**Corollary (Minimum Codomain Structure).** A hallucination-free language model requires:
```
π_k(C) ⊇ π_k(Sem)   for all k ≥ 0.
```
The minimal such C is the ∞-groupoid completion of Sem itself.

### T5. Hallucination–Homology Correspondence (Source 1 Theorem 4.1; Source 2 Theorem 3.1)

**Theorem (Main Theorem).** Let σ = Σᵢ nᵢ σᵢ ∈ C_n(K_•) be an n-chain of claims generated by a language model. Then:
```
σ is hallucinated ⟺ [σ] ≠ 0 ∈ H_n(K_•; Z)
```
A chain of claims is hallucinated if and only if it represents a non-trivial homology class.

**Proof (⟸):** Suppose [σ] ≠ 0. Then σ ∈ Z_n \ B_n: it is a cycle (∂_n σ = 0, claims are internally consistent at boundaries) but not a boundary (σ ∉ im(∂_{n+1}), no (n+1)-chain of justifications has σ as its boundary). This is the defining characteristic of hallucination: locally consistent but globally unfounded.

**Proof (⟹):** Suppose σ is hallucinated. Two cases: (1) If ∂_n σ ≠ 0: claims are not even internally consistent — detectable at (n-1)-level. (2) If ∂_n σ = 0: the claims form a cycle. By the hallucination hypothesis, no τ ∈ C_{n+1} exists with ∂_{n+1}τ = σ, so σ ∉ B_n and [σ] ≠ 0.

**Corollary (Acyclicity = Hallucination-Freedom).** If H_n(K_•; Z) = 0 for all n ≥ 1, then every internally consistent chain of claims has a justification. No hallucination of "locally plausible but globally unfounded" type is possible.

### T6. The Hallucination–Homotopy Correspondence (HHC) — Full Classification Theorem (Source 1 Theorem 5.1; Source 2 Theorem 4.1)

**Theorem.** The following is a complete classification of hallucination types by homotopy-theoretic invariants:

| Hallucination Type       | Description                                          | Invariant    | Obstruction              |
|--------------------------|------------------------------------------------------|--------------|--------------------------|
| Unjustified inference    | Claiming B from A with no connecting path            | π₀           | Disconnection            |
| Circular reasoning       | Self-reinforcing loop of claims                      | π₁ ≠ 0       | Non-contractible loop    |
| Inconsistent justifications | Two incompatible proofs of same claim            | π₂ ≠ 0       | Incoherent 2-cell        |
| Fabricated entity chain  | Consistent-seeming chain with no filling             | H_n ≠ 0      | Homological hole         |
| Compositional drift      | Context shift changes meaning silently               | Holonomy ≠ id| Transport anomaly        |

**Proofs of each case:**

**Case 1 (Unjustified inference / π₀):** A claim B is asserted from A, but A and B lie in different connected components: [A] ≠ [B] in π₀(Sem), so ||A =_Sem B|| = 0. No justification path exists. An LLM generates such inferences because F(A) and F(B) may be close in R^d (high cosine similarity) despite being in different semantic components.

**Case 2 (Circular reasoning / π₁):** A loop of inferences c₀ → c₁ → ··· → c_k → c₀ defines [γ] ∈ π₁(Sem, c₀). If [γ] ≠ e, the loop has no 2-disk filling: circular reasoning without independent grounding. By Hurewicz, a non-trivial π₁ element maps to a non-trivial element of H₁.

**Case 3 (Inconsistent justifications / π₂):** Two paths p, q: a → b represent two justifications for the same conclusion. If π₂(Sem, a) ≠ 0, there exist p, q with no 2-cell connecting them: genuinely incompatible justifications. An LLM conflating them (implicitly assuming p = q) hallucinates at the 2-cell level.

**Case 4 (Fabricated entity chain / H_n):** A chain σ ∈ C_n(K_•) with ∂_n σ = 0 but [σ] ≠ 0 ∈ H_n is a cycle of mutually consistent but collectively unfounded claims. Each individual claim seems reasonable, each adjacent pair is consistent, but the construction has no basis. This is Theorem T5 itself.

**Case 5 (Compositional drift / Holonomy):** Consider a : A(γ) in the fiber over context γ, and a loop ℓ in context space B based at γ. Parallel transport along ℓ yields transport_ℓ(a) : A(γ). If transport_ℓ(a) ≠ a — the holonomy is non-trivial — then the claim silently changes meaning after a round trip of context modifications. The holonomy group Hol(∇, γ) ⊆ Aut(A(γ)) of the semantic connection ∇ classifies this failure mode.

### T7. Functoriality of the HHC (Source 2 Theorem 4.2)

**Theorem.** There exists a functor:
```
Φ: Hall → hTop_*
```
from the category of hallucination events to the category of pointed homotopy types, mapping:
- A hallucination event (M, Γ, â, A) to the pointed homotopy type (Σ_σ, [σ]) where σ is the corresponding cycle/obstruction and [σ] is its homotopy class.
- A morphism of hallucination events to the induced map on homotopy types.

Moreover, Φ is:
1. **Faithful:** distinct hallucination events map to distinct homotopy-theoretic obstructions.
2. **Reflects isomorphisms:** if Φ(h) ≃ Φ(h'), then h and h' are structurally equivalent hallucinations.

**Proof sketch.** Φ is constructed by composing: (i) extraction of the claim chain σ from the hallucination event; (ii) computation of the homology/homotopy class [σ] in K_•; (iii) Hurewicz mapping to the pointed homotopy type. Faithfulness follows from distinct hallucination events producing distinct chains. Isomorphism reflection follows from the universality of the Hurewicz homomorphism.

### T8. Existence of Hallucinations (Inverse Problem) (Source 2 Prop 4.1)

**Proposition.** For every non-trivial homotopy class [α] ≠ 0 ∈ π_n(Sem, a) or homology class [σ] ≠ 0 ∈ H_n(K_•), there exists a prompt and context that can elicit the corresponding hallucination from a sufficiently capable LLM operating in Vect.

**Proof.** The embedding functor F maps the non-trivial class [α] to the trivial class F_*([α]) = 0 ∈ π_n(Vect) = 0. The LLM "sees" a contractible path where a topological obstruction exists. By providing a context Γ near the support of the cycle α, the LLM will generate a completion traversing the hole.

### T9. Non-Faithfulness Proposition (Source 1 Prop 2.1)

**Proposition.** F is not faithful: distinct morphisms f, g: Γ → Δ in Sem can satisfy F(f) = F(g) in Vect_R. Proof: two distinct justifications for the same conclusion map to the same embedding region.

### T10. Statistical Similarity ≠ Semantic Equivalence (Source 1 Prop 6.1; Source 2 Theorem 6.1)

**Theorem.** Let cos(F(A), F(B)) ≥ 1 − ε for embeddings F(A), F(B) ∈ R^d. This does NOT entail the existence of maps f, g, η, ε, τ satisfying the full equivalence data.

**Proof.** Cosine similarity is a metric condition: d(F(A), F(B)) < δ. Semantic equivalence is an algebraic condition requiring explicit maps and coherence data at all levels. Example: "the morning star" and "the evening star" have maximal embedding similarity (both reference Venus) yet the identity type carries non-trivial astronomical content (Frege's puzzle). The coherence condition τ requires that the two "directions" of identification are compatible — invisible to any metric. More formally: the space of embeddings within ε of F(A) is contractible (a ball), but the space of semantic equivalences A ≃ B is generically non-contractible (non-trivial π₁ corresponding to distinct auto-equivalences).

### T11. Attention as Approximate Section (Source 2 Prop 2.2)

**Proposition.** The attention mechanism computes an approximate section s: B → E of the semantic fibration: for each context γ, it produces s(γ) ∈ E intended to lie in p⁻¹(γ). But attention computes this in Vect (softmax-weighted sums), providing no guarantee that the result lies in the correct fiber.

### T12. Failure of Vectorial Attention (Source 2 Prop 8.1)

**Proposition.** Standard softmax attention:
```
Att_Vect(Q, K, V) = softmax(QK^T / √d) V
```
computes a weighted colimit in Vect (weighted sum). When the diagram D is incoherent (mutually contradictory claims), the weighted colimit in Vect still exists, but the weighted limit in Sem does not. Current LLMs produce output even from contradictory contexts.

**Proof.** In Vect, colimits are quotients of direct sums, which always exist. In Sem, limits require coherence. When context contains A and ¬A, no cone exists over the diagram, so the limit is empty. Generating output in this case = term of an empty type = hallucination.

### T13. Hallucination as Sheaf Obstruction (Source 2 Theorem 7.1)

**Theorem.** A generated claim ŝ is hallucinated iff the obstruction class:
```
o(ŝ) ∈ H¹(K_•, F)
```
is non-trivial: local sections (individually locally consistent claims) fail to glue into a global section (a globally coherent narrative).

**Proof.** From the sheaf cohomology exact sequence:
```
0 → H⁰(K_•, F) → Πᵢ F(Uᵢ) →^δ Πᵢ<ⱼ F(Uᵢ ∩ Uⱼ) → H¹(K_•, F) → …
```
H⁰ = global sections (globally consistent claims); H¹ classifies obstructions to gluing local sections. A hallucinated claim provides local sections that fail the gluing condition.

**Corollary (Duality).** H¹(K_•, F) ≠ 0 is dual to H₁(K_•) ≠ 0 (by the universal coefficient theorem). Homology detects "holes" (missing justifications); cohomology detects "obstructions" (failures of local-to-global consistency).

### T14. Persistent Hallucination Detection (Source 1 Theorem 8.1; Source 2 Theorem 9.1)

**Theorem.** Let {v_t}_{t=1}^T be the embedding sequence during text generation. Compute the Vietoris–Rips filtration on {v_t} ∪ S_K (knowledge base embeddings). If there exists a persistent class [σ] ∈ H_n with persistence > θ such that the generated path traverses the support of [σ], then the generated text is hallucinating with confidence proportional to the persistence.

**Proof sketch.** A persistent H₁ class corresponds to a stable loop in knowledge embedding that cannot be filled at any scale up to δ (persistence δ − β > θ). If the generation trajectory crosses this loop, it traverses "empty semantic space" where no grounded knowledge provides support. The persistence threshold θ filters noise-induced ephemeral features from genuine topological structure, following the stability theorem of persistent homology (Edelsbrunner & Harer 2010).

### T15. Knowledge Extension Long Exact Sequence (Source 2 Prop 3.1)

**Proposition.** Let K_• ↪ L_• be an inclusion of knowledge complexes. The long exact sequence:
```
… → H_n(K_•) → H_n(L_•) → H_n(L_•, K_•) → H_{n-1}(K_•) → …
```
describes how adding knowledge can fill existing holes (killing elements of H_n(K_•)) or create new holes (via relative homology H_n(L_•, K_•)). The connecting homomorphism ∂_*: H_n(L_•, K_•) → H_{n-1}(K_•) identifies which new hallucination risks are introduced by knowledge extension.

### T16. Circular Reasoning as H₁ Class (Source 2 Example 3.1)

**Example.** Three claims c₀, c₁, c₂ with justifications j₀₁: c₀ → c₁, j₁₂: c₁ → c₂, j₂₀: c₂ → c₀ but no 2-simplex (c₀, c₁, c₂) filling the triangle. The chain σ = [c₀,c₁] + [c₁,c₂] + [c₂,c₀] is a 1-cycle (∂₁σ = 0) but not a boundary, so [σ] ≠ 0 ∈ H₁(K_•). This represents circular reasoning: each step seems justified, but the circle as a whole has no independent grounding.

### T17. Univalence Axiom (Source 1 Axiom 6.1; Source 2 Axiom 6.1)

For types A, B : U, the canonical map (A =_U B) → (A ≃ B) is itself an equivalence:
```
(A ≃ B) ≃ (A =_U B)
```

The Univalence Constraint as hallucination filter: a model satisfying univalence would only identify two concepts when the full equivalence data (maps + coherence) exists, not merely when embeddings are close. This eliminates entity confusion, attribute transfer, and analogical overextension.

## The 5 Generation Principles (HoTT-LM Architecture)

Articulated in Source 1 §10 and Source 2 §11 under "Design Principles":

**Principle 1 — Generation as Type Inhabitation:**
Replace argmax_t P(t | ctx) with the search problem: find a such that Γ ⊢ a : A has a derivation δ, minimizing a cost function C(a, δ) over valid inhabitants. The constraint Γ ⊢ a : A ensures type-correctness.

**Principle 2 — Certified Derivations:**
Every generated term a : A comes with a derivation tree δ witnessing Γ ⊢ a : A, checkable in polynomial time.

**Principle 3 — Abstention on Empty Types:**
When A is uninhabited (no valid claim of this type exists given the context), the model returns ⊥ rather than generating a hallucinated term. This is the categorical equivalent of a type error at generation time.
```haskell
hottGenerate model ctx goalType =
  case searchDerivation ctx goalType of
    Just (term, deriv) ->
      case typeCheck ctx term goalType of
        Right True -> Just (term, deriv)
        _          -> Nothing  -- ABSTAIN
    Nothing -> Nothing         -- ABSTAIN
```

**Principle 4 — Context as Fibration:**
The context window is a dependent telescope (not a flat token sequence), with each position typed by a semantic type that may depend on preceding positions.

**Principle 5 — Attention as Limit:**
The attention mechanism computes a weighted limit in Sem (when coherent) or signals incoherence (when contradictory), rather than a weighted sum in Vect that silently blends contradictions.

## The Type-Checking Algorithm

Core function (Hs:VII; Source 1 §9; Source 2 §10):
```haskell
typeCheck :: Context -> SemTerm -> SemType -> Either HallucinationType Bool
typeCheck ctx (Grounded claim src) (Prop p)
  | claim == p = Right True          -- Exact match with source
  | otherwise  = Left (TypeMismatch claim p)
typeCheck ctx (Inferred base (Statistical prob)) ty
  = Left (StatisticalNotProof prob)  -- Probability is NOT a derivation
typeCheck ctx (Inferred base (Entailment a b)) ty
  = typeCheck ctx base ty            -- Entailment preserves type
typeCheck ctx (PathWitness a b NoEvidence) (PathType ty _ _)
  = Left (MissingPathDerivation a b) -- No evidence = hallucination
typeCheck ctx (Hallucinated _) _
  = Left FreeFloating                -- Always fails
typeCheck ctx (Refl a) (PathType ty x y)
  | termEq a x && termEq a y = Right True
  | otherwise = Left (ReflMismatch a x y)
```

## Topic Overviews

### Topic 1: Topological Hallucination Detection

**Background:**
The insight is that semantic space Sem is an ∞-groupoid (a space with non-trivial homotopy groups at every dimension), while LLMs operate in Vect_R (contractible, all π_n = 0). Hallucination arises precisely in the gap between these two structures.

**Key results:**
1. Hallucination ⟺ non-trivial homology class [σ] ≠ 0 ∈ H_n(K_•) (T5)
2. Five hallucination types correspond bijectively to five topological invariants (T6)
3. The correspondence is functorial via Φ: Hall → hTop_* (T7)
4. No faithful functor Sem → Vect exists (T4 — structural inevitability)
5. Hallucination ⟺ H¹(K_•, F) ≠ 0 (sheaf-cohomological dual formulation) (T13)
6. Persistent hallucination detection via Vietoris-Rips filtration and persistence barcodes (T14)

**The five cases in detail (for Paper 1):**
- π₀ (unjustified inference): A and B in different connected components; F(A) close to F(B) in Vect but semantically disconnected
- π₁ (circular reasoning): non-contractible loop; H₁ detection via 1-cycles c₀→c₁→c₂→c₀ (T16)
- π₂ (inconsistent justifications): two paths p,q: a→b with no 2-cell connecting them
- H_n (fabricated entity chain): cycle with ∂_n σ = 0 but σ ∉ B_n (the main theorem T5)
- Holonomy (compositional drift): transport_ℓ(a) ≠ a for loop ℓ in context space

**Practical detection algorithm (Hybrid Architecture):**
1. Generate candidate tokens via standard next-token prediction.
2. For each candidate, compute embedding and check:
   - Does trajectory traverse a persistent H₁ class? (persistence check)
   - Does candidate complete a non-bounding cycle? (simplicial homology check)
   - Does candidate type-check against inferred goal type? (type filter)
3. Reject candidates failing topological checks.
4. If all fail: abstain or request clarification.

**Computational feasibility:**
- Persistent homology: O(m³) via standard reduction; Ripser achieves near-linear on sparse inputs.
- Knowledge base barcode can be precomputed; incremental updates during generation.
- Full type checking is undecidable; restrict to n-truncated types for bounded n.

**Open problems:**
- Empirical validation: hallucination frequency should correlate with Betti numbers of the local knowledge complex.
- Automating construction of K_• from raw corpora (co-consistency judgments are themselves uncertain).
- Approximate persistent homology methods (witness complexes, landmark-based) for production scale.

**Relevant frameworks:**
- Algebraic topology: simplicial homology, singular homology, Hurewicz theorem
- Persistent homology: Vietoris-Rips filtration, persistence barcodes, stability theorem
- Sheaf cohomology: Grothendieck topologies, gluing, H¹ obstruction classes

### Topic 2: Type-Theoretic Generation (The 5 Generation Principles)

**Background:**
Current LLMs compute argmax P(token | context), where P is a measure on embedding space. This has no notion of "type" or "proof." The HoTT-LM replaces this with type inhabitation search: find a : A with derivation δ witnessing Γ ⊢ a : A.

**The 5 Principles (full specification):**

**P1 — Generation as Type Inhabitation:**
- Old: argmax_t P(t | ctx)
- New: find a such that Γ ⊢ a : A, minimizing cost C(a, δ) over valid inhabitants
- Implementation: proof search (focusing, backchaining, unification)
- Reference: D23 (HoTTLM.generate), Hs:VIII

**P2 — Certified Derivations:**
- Every generated term a : A comes with Derivation δ witnessing Γ ⊢ a : A
- Derivation types: AxiomD (sourced), IntroD, ElimD, PathIntroD, TransportD, UnivalenceD
- Derivation is checkable in polynomial time (forward verification)
- Reference: D7 (Derivation), D24 (Judgment)

**P3 — Abstention on Empty Types:**
- When A is uninhabited: return Nothing (⊥) rather than hallucinating
- This is the categorical equivalent of a type error at generation time
- Implementation: `hottGenerate` returns Nothing on both search failure and typecheck failure
- Reference: D23 (HoTTLM), Hs:VIII (hottGenerate)
- Key contrast: Vect always has elements; there is no way to represent "empty type" in Vect

**P4 — Context as Fibration:**
- Context window is a dependent telescope: (x₁:A₁), (x₂:A₂(x₁)), …
- Each position's type can depend on all preceding positions
- The LLM's P(token | context) is a measure on the fiber p⁻¹(γ)
- Hallucination = measure assigns mass outside the fiber
- Reference: D8 (Context), D11 (Semantic Fibration), Hs:IV

**P5 — Attention as Limit:**
- Old: Att_Vect(Q,K,V) = softmax(QK^T/√d)V (weighted colimit in Vect, always exists)
- New: Att(D, W) = lim^W D (weighted limit in Sem, may not exist)
- When diagram D is incoherent (contradictions in context): limit does not exist → signal incoherence → do not generate
- Implementation: `categoricalAttend` returns `Nothing` when diagram is incoherent
- Reference: D28 (CategoricalAttention), Hs:X, T12 (Failure of Vectorial Attention)

**Type-checking judgment Γ ⊢ a : A:**
The core judgment replacing next-token prediction. A claim is valid iff it type-checks, not iff it is probable (D24). The `typeCheck` function implements this with exhaustive case analysis (see The Type-Checking Algorithm section above).

**Key type-system distinctions:**
- `Grounded` term (sourced) vs `Hallucinated` term (no source)
- `Entailment` justification (genuine 1-cell) vs `Statistical Float` (no path structure)
- `PathWitness with Definitional/Propositional/ByUnivalence` (valid evidence) vs `PathWitness with NoEvidence` (empty path type)
- Inhabited type (can generate) vs uninhabited type (must abstain)

**Relationship to proof assistants:**
The HoTT-LM is essentially a proof assistant augmented with neural candidate generation. The neural component proposes candidates; the type-theoretic component verifies and filters them. Existing tools: Agda, Coq, Lean, Idris implement similar type-checking logic.

**Open problems:**
- Type inhabitation for full homotopy types is undecidable. Restrict to:
  - n-truncated types for bounded n (decidable for sets and groupoids)
  - Proof search strategies adapted from ATP (focusing, backchaining, unification)
  - Caching common derivation patterns as "proof libraries"
- The learning problem: how does a model learn the homotopy structure of Sem from data?
- Can persistent homology of training data embeddings serve as a training signal?

### Topic 3: HoTT Synthesis — Unification via Detection-Generation Duality

**Background:**
Papers 1 and 2 establish two perspectives on the same mathematical structure:
- **Detection:** given a generated claim, check if it corresponds to a non-trivial homology class (hallucination detection).
- **Generation:** given a semantic type, search for a valid inhabitant (hallucination prevention).

These are dual operations on the same semantic ∞-groupoid.

**The Detection-Generation Duality:**

| Detection (Paper 1/2) | Generation (HoTT-LM) | Semantic Structure |
|-----------------------|---------------------|-------------------|
| Check [σ] ≠ 0 in H_n | Confirm Γ ⊢ a : A | Knowledge Complex K_• |
| Find TopologicalObstruction | Construct Derivation | Path structure of SemType |
| Check π₀ disconnection | Verify Grounded term reachable | Connected components |
| Find non-contractible loop | Verify no circular derivation | π₁(Sem) |
| Check incoherent 2-cell | Verify PathEvidence coherence | π₂(Sem) |
| Persistent homology scan | Type inhabitation search | Vietoris-Rips filtration |

**The Univalence Constraint as the bridge:**
The Univalence Axiom (T17) provides the constraint that unifies detection and generation:
- For detection: two concepts should only be identified when a full semantic equivalence A ≃ B exists (not just cosine similarity).
- For generation: when generating a term of type A, using a term of type B (≃ A) is valid iff the full equivalence data is available.

**The HHC Functor Φ as the synthesis map:**
The functor Φ: Hall → hTop_* (T7) establishes that the classification is complete and the two perspectives are equivalent:
- Every hallucination event maps to a topological obstruction.
- Every topological obstruction maps back to a potential hallucination (T8, inverse problem).
- The mapping is faithful (no information lost) and reflects isomorphisms.

**Sheaf cohomology connection:**
- H₁(K_•) ≠ 0 (homology, detection perspective) ↔ H¹(K_•, F) ≠ 0 (cohomology, generation/obstruction perspective).
- These are dual by the universal coefficient theorem (T13 corollary).
- The sheaf gluing condition is exactly the coherence condition violated by hallucination.

**Categorical architecture for synthesis:**
```
Sem ←——— K_• ——→ Vect
(∞-Grpd)  (complex)  (contractible)
   |                     |
   | Full                | Approximate
   |                     |
HoTT-LM              Current LLM
(type inhabitation)  (next-token prediction)
```

The knowledge complex K_• is the intermediary that makes the connection between the ∞-groupoid structure of Sem and the practical embedding-based methods in Vect computable.

**The Yoneda Perspective (Source 2 Remark 2.1):**
By the Yoneda lemma, an object A in category C is completely determined by Hom_C(-, A): C^op → Set. Applied to semantic types: the meaning of a concept is entirely determined by its relationships to all other concepts. An embedding F(A) ∈ R^d discards morphism-level structure, retaining only a compressed summary of the Hom-sets.

This is the categorical formulation of Wittgenstein's "meaning is use" and Firth's distributional hypothesis, but with the addition that morphisms themselves carry structure (path types, higher homotopies).

**Presheaf models and Kripke-Joyal semantics (Source 2 §13.3):**
In a presheaf model PSh(I) over index category I (of contexts), internal logic is Kripke-Joyal semantics where truth is context-dependent: i ⊩ φ ⟹ ψ iff for all f: j → i, j ⊩ φ implies j ⊩ ψ. Current LLMs compute flat conditional probability P(token | ctx) rather than Kripke-forced truth i ⊩ φ, losing the universal quantification over context refinements.

**Cubical type theory connection (Source 2 §13.2):**
Computational content is best realized in cubical type theory (Cohen et al. 2017), where paths are computed as functions from the interval [0,1] (or De Morgan algebra) rather than axiomatized. A cubical path p: I → A with p(0) = a, p(1) = b is a concrete justification, not merely an assertion of equality. Cubical operations (composition, filling, coercion) provide the computational machinery for justified inference.

**Connection to AI memory architectures (Source 2 Remark 7.1):**
The sheaf formulation suggests typed memory schemas defining a sheaf over the temporal poset of conversation states. A valid memory state = global section; type-checking violations = cohomological obstructions detected at write time. Li (2025) provides the dynamical counterpart: memory traces as nontrivial cycles [γ] ∈ H₁(Z) on a latent cognitive manifold.

## Architecture Summary: HoTT-LM Components

**Four interacting subsystems (Source 1 §10.2; Source 2 §11):**

1. **Type Inference Engine:** Given context Γ and partial generation, infer semantic type A of the next claim. Replaces vocabulary prediction with type prediction.

2. **Inhabitation Search:** Given Γ and A, search for a : A using proof-search strategies (focusing, backchaining, unification). Returns Nothing when type is uninhabited.

3. **Coherence Checker:** Verify that generated term's derivation is coherent with all existing paths in context — no topological obstruction is violated.

4. **Homological Monitor:** Continuously compute persistent homology of growing embedding trace; flag when trajectory approaches a non-trivial homology class.

## Gaps and Research Directions for Worker Notes

1. **Empirical validation gap:** Neither paper provides empirical results correlating Betti numbers of K_• with hallucination rates. This is flagged as the most urgent next step (Source 2 §13.4). Workers should search for 2025–2026 TDA papers on LLM hallucination.

2. **Knowledge complex construction:** How to automatically build K_• from raw text is unaddressed. Co-consistency judgments require themselves a type-theoretic framework. Workers should review knowledge graph extraction literature.

3. **Training signal from persistent homology:** The question of whether PH of training embeddings can serve as a training signal is explicitly open (Source 2 §13.4).

4. **Cubical HoTT implementation:** The concrete computational content (cubical paths as functions I → A) is discussed but not implemented. Agda/Cubical and Lean 4 are the relevant tools.

5. **The n-truncation tradeoff:** For n = 0 (sets), type checking is decidable and captures the π₀ and H_n cases. For n = 1 (groupoids), it captures π₁ as well. Full ∞-groupoid type checking is undecidable. The boundary of decidable detection is not characterized.

6. **Holonomy computation:** The compositional drift case (Case 5 of HHC) is the least developed. The holonomy group Hol(∇, γ) ⊆ Aut(A(γ)) is defined but no algorithm is given for computing or detecting it.

7. **Agent swarm extension:** Source 1 §11.3 sketches multi-agent hallucination monitoring but does not formalize it. This is a direct extension opportunity.

8. **Li (2025) connection:** The reference to Li (arXiv:2508.11646) on memory as structured trajectories / persistent homology and contextual sheaves appears only in Source 2. Workers should read and cite this paper as it provides the dynamical / temporal counterpart to the static framework here.

9. **The Yoneda constraint:** The remark that meaning = totality of Hom-sets (D2 / Remark 2.1 in Source 2) is suggestive but not developed into a theorem. This is an opportunity for Paper 3.

10. **Relation to attention mechanism:** Categorical attention (Principle 5 / T12) reframes softmax attention as a failed limit computation. A concrete alternative attention algorithm implementing `categoricalAttend` is not given.
